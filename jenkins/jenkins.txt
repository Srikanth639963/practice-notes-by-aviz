Build Trigger Options :

Build Manually : You manually trigger the build to run the job.

Build periodically : Runs on a particular interval. This uses cron syntax.

Poll SCM : Periodically checks the repo for code changes based on given interval, If any change occured, then it triggers the build.


Github Webhook : No periodical checks, Pipeline triggers based on commit/push happend on the git repo.

PayloadURL: http://3.110.98.198:8080/github-webhook/

Maven : Maven is a build and project management tool, primarily used in java based projects.

Dependency Management : Automatically fetches libraries and their sub depends (Transitive Dependencies)
Build Management : Compile, test and package
Standardised Structure : All project structures looks same
free & opensource : 
Reprodusable build: same output across the machines.
plugin ecosystem : Hundred of the pulgins avaibale in market.

Java: maven, ant, gradle
js : npm
python: pybuilder
php: composer
c/c++ : make

JAR: Java ARchieve : A JAR file bundles compiled java .class files and its metadata into a single compressed file for distribution.
JAR includes the compiled .class files of a Java project.

WAR : Web Application Archieve : This packages a complete Java Web applicaiton for deployment on a webserver i.e; apache tomcat.
A WAR file includes all the components of a web application, such as servlets, JavaServer
Pages (JSPs), HTML files, JavaScript files, and images.


Step 1 : Developer writes app.java (source code)
Step 2 : Javac compiles app.java --> app.class (bytecode)
Step 3 : Maven packages all .class files --> myapp.jar or myapp.war
Step 4 : WAR is deployed to apache tomcat --> users can access it via browser.

For example, to add the manager-gui role to a user named tomcat with a password of s3cret, add the following to the config file listed above.

<role rolename="manager-gui"/>
<user username="tomcat" password="s3cret" roles="manager-gui"/>
Note that for Tomcat 7 onwards, the roles required to use the manager application were changed from the single manager role to the following four roles. You will need to assign the role(s) required for the functionality you wish to access.

manager-gui - allows access to the HTML GUI and the status pages
manager-script - allows access to the text interface and the status pages
manager-jmx - allows access to the JMX proxy and the status pages
manager-status - allows access to the status pages only


<role rolename="manager-gui"/>
<role rolename="manager-script"/>
<role rolename="manager-jmx"/>
<role rolename="manager-status"/>
<role rolename="admin-gui"/>
<user username="tomcat" password="Avinash" roles="manager-gui,manager-script,manager-jmx,manager-status,admin-gui"/>
