

CI/CD : 

CI : Continuous integration : Dev team freq merge code changes into a shared repo, where automation starts..

CD : Continuous Delivery : Code changes are picked, automatically build, tested and preapred for release.. A manual step/approval involved before the deployment.

CD : Continuous Deployment : Code changes are picked, automatically build, tested and deploymenyed automatically.

Build Trigger Options :

Build Manually : You manually trigger the build to run the job.

Build periodically : Runs on a particular interval. This uses cron syntax.

Poll SCM : Periodically checks the repo for code changes based on given interval, If any change occured, then it triggers the build.


Github Webhook : No periodical checks, Pipeline triggers based on commit/push happend on the git repo.

PayloadURL: http://3.110.98.198:8080/github-webhook/

Number of executors : The maximum number of concurrent builds that Jenkins may perform on this node.

Throttle builds at job level : Enforces a minimum time between builds based on the desired maximum rate.
Note: this does not enforce an "average" rate, it only looks at the time since last build.
