--> Always use an IDE. \

--> terraform init		: initiaze the directory
--> terraform validate	: validate the code syntax errors
--> terraform fmt		: fomats the traaform code (proper indentations)
--> terraform plan		: Similar to dryrun. THis displays what resources it is going to create/modify/delete.
terraform apply			: Create the resources

terraform apply -auto-approve

---

replace (formarly taint) : helps to recreate the resources (if it currupts).. 
It destroys existing resources and then create a brand new resource.. State file updates with new resource info..

terraform apply -replace="aws_instance.mumbai-server"

--

This command identifies the changes happend outside of the terraform : terraform plan -refresh-only

This command update the statefile with current state of the resource: terraform apply -refresh-only

---

terraform plan -refresh-only		--> Detect the change
terraform apply						--> Ignore the changes and bring resource to actual state
terraform apply -refresh-only		--> If chnages valid, updates the tfstate file with actual changes

Terraform Statefile: Source of truth. This file contains all the resources and its state managed by terraform.

terraform state list

terraform state show aws_s3_bucket.mybucket

terraform init -migrate-state

terraform force-unlock <ID>	(Emergency cases)

provider version Constraints:


terraform init -upgrade				--> Run this after specifying the provider version.



terraform {
  required_providers {
    aws = {
      source = "hashicorp/aws"
      version = "6.31.0"
    }
  }
}



= 5.0.0			--> Exactly same version
>= 5.0			--> Either 5 or higher
<= 5.5			--> lower than 5.5 or 5.5
~> 5.0			--> 5.x only


=======

When passing Environment varibales, This is the flow that will have priority

1. CLI Flag			--> terraform apply -var="enable_stop_protection=true"
2. .tfvars file		--> terraform apply -var-file="dev.tfvars"
3. Environment var	--> export TF_VAR_instance_type="t3.micro"
4. defualt value	--> defined inside variables block

workspaces : Each workspace maintains its own statefiles.

terraform workspace list

terraform workspace new dev
terraform workspace new uat
terraform workspace new prd


terraform workspace select default

terraform workspace select dev
terraform state list
terraform apply -var-file="dev.tfvars" -auto-approve
terraform state list


terraform workspace select uat
terraform apply -var-file="uat.tfvars" -auto-approve

terraform workspace select prd
terraform apply -var-file="prd.tfvars" -auto-approve

terraform destroy -var-file="prd.tfvars" -auto-approve


terraform workspace select uat
terraform destroy -var-file="uat.tfvars" -auto-approve

terraform workspace select dev
terraform destroy -var-file="dev.tfvars" -auto-approve

terraform workspace list
terraform workspace select default

terraform workspace delete dev
terraform workspace delete uat
terraform workspace delete prd
